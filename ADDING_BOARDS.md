# Adding a New Board

Guide for adding a new board to the lab manuals and PDF generation pipeline.

Replace `<board>` throughout with your board's short identifier (e.g. `same54`, `pic32bz6`).

---

## 1. Create Source Files (`lab_manuals/src/`)

```bash
mkdir -p lab_manuals/src/<board>
```

**Required files:**

- `index.md` — Board overview (hardware requirements, board description)
- `lab3.md` — Board-specific lab

**Optional:**

- `additional-software-setup.md` — Any board-specific software setup steps

**`index.md` frontmatter:**

```yaml
---
title: "<Board Name> - Board Overview"
sidebar_label: Overview
sidebar_position: 1
slug: /docs/<board>
---
```

**`lab3.md`** imports the shared overlay files for common content and adds board-specific steps in between. The overlay files live in `shared/` but must be symlinked into the board directory so the relative imports resolve:

```bash
cd lab_manuals/src/<board>
ln -s ../shared/_lab3-pre-overlay.md _lab3-pre-overlay.md
ln -s ../shared/_lab3-post-overlay.md _lab3-post-overlay.md
```

```mdx
---
title: "Lab 3: ..."
sidebar_label: "Lab 3: ..."
sidebar_position: 5
---

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import PreOverlay from './_lab3-pre-overlay.md';
import PostOverlay from './_lab3-post-overlay.md';

# Lab 3: ...

<PreOverlay />

<!-- Board-specific steps here -->

<PostOverlay />
```

**Images:** place board-specific images in `lab_manuals/static/images/<board>/` and reference them as `/images/<board>/filename.png`.

## 2. Board Variables

Board variables (`%VAR%`) must be added in **two places** — keep them in sync.

### `gh_pages/plugins/remarkBoardVars.mjs`

```javascript
const boardMap = {
  // ... existing boards ...
  <board>: {
    BOARD:            '<zephyr-board-id>',   // e.g. 'same54_xpro'
    BOARD_NAME:       '<Human Readable Name>',
    FLASH_ARGS_LINUX: '',                    // extra west flash args, or ''
    FLASH_ARGS_MACOS: '',
    FLASH_ARGS_WIN:   '',
  },
};
```

### `pdf-gen/plugins/boardVars.mjs`

Same entry in the `boardMap` export in this file.

---

## 3. PDF Generator (`pdf-gen/generate.mjs`)

Add an entry to `PAGE_SEQUENCE`:

```javascript
const PAGE_SEQUENCE = {
  same54: [ /* existing */ ],
  pic32bz6: [ /* existing */ ],
  <board>: [
    'home.md',
    '<board>/index.md',
    '<board>/additional-software-setup.md',  // omit if not needed
    'shared/lab1.md',
    'shared/lab2.md',
    '<board>/lab3.md',
    'appendices/appendix-a.md',
    'appendices/appendix-b.md',
    'appendices/appendix-c.md',
    'appendices/appendix-d.md',
  ],
};
```

Test:

```bash
cd pdf-gen
node generate.mjs --board <board> --os linux
```

Output: `lab_manuals/pdf/<board>/Zephyr-Getting-Started_<board>_linux.pdf`

---

## 4. Docusaurus (`gh_pages/`)

### `gh_pages/sidebars.js`

```javascript
const sidebars = {
  same54Sidebar:     [{type: 'autogenerated', dirName: 'same54'}],
  pic32bz6Sidebar:   [{type: 'autogenerated', dirName: 'pic32bz6'}],
  <board>Sidebar:    [{type: 'autogenerated', dirName: '<board>'}],  // add this
  appendicesSidebar: [{type: 'autogenerated', dirName: 'appendices'}],
};
```

### `gh_pages/docusaurus.config.js`

Add a navbar item in the `navbar.items` array:

```javascript
{
  type: 'docSidebar',
  sidebarId: '<board>Sidebar',
  label: '<BOARD LABEL>',
  position: 'left',
},
```

Test:

```bash
cd gh_pages
pnpm start
```

Navigate to the new board section and verify variable substitution and content.

---

## 5. Commit & Deploy

1. Commit source changes and push to `main`
2. GitHub Actions builds and deploys the site to the `gh-pages` branch automatically
3. Commit any generated PDFs from `lab_manuals/pdf/<board>/`
