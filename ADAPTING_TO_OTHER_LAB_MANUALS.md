# Adapting This Toolchain for Your Own Lab Manual

This repo contains two independent publishing tools:

- **`pdf-gen/`** — generates printable PDFs from Markdown source
- **`gh_pages/`** — publishes a Docusaurus website to GitHub Pages

Both tools read content from `lab_manuals/markdown/`. You can use either one, or both.

---

## What You Need to Change

| Tool | Files to edit |
|------|---------------|
| PDF generator | `pdf-gen/config.mjs` only |
| Website | `gh_pages/docusaurus.config.js` + `gh_pages/sidebars.js` |

Everything else — the rendering pipeline, plugins, stylesheet — requires no changes.

---

## Part 1: PDF Generator

### Step 1 — Edit `pdf-gen/config.mjs`

This is the only file you need to touch. Open it and update four things:

**1. Cover page text**

```js
cover: {
  title:        'My Lab Manual Title',
  organization: 'Microchip Technology',
},
```

**2. Output filename base**

```js
outputName: 'my-lab-manual',
// Produces: lab_manuals/pdf/<board>/my-lab-manual_<board>_<os>.pdf
```

**3. Versions (boards and OSes)**

If you have one board and don't need per-OS variants:

```js
versions: {
  boards: {
    myboard: {
      displayName: 'My Board Name',  // shown on cover page
      // omit "vars" if you don't use %TOKEN% substitution
    },
  },
  os: ['default'],  // single entry = one PDF per board
},
```

If you have multiple boards or OS variants, add more entries:

```js
versions: {
  boards: {
    boardA: { displayName: 'Board A' },
    boardB: { displayName: 'Board B' },
  },
  os: ['linux', 'macos', 'windows'],
},
```

**4. Intro file and page sequence**

`introFile` is prepended to every PDF. It resolves from `lab_manuals/markdown/` root.

`pages` lists the remaining pages in order:
- **Bare filename** (no `/`) → looked up in `lab_manuals/markdown/<board>/`. If the file does not exist for a given board it is silently skipped, so one list works for all boards.
- **Path with a slash** → resolved from `lab_manuals/markdown/` root. Use this for appendices or other shared content that lives outside the board folder.

```js
introFile: 'home.md',
pages: [
  'index.md',                  // → <board>/index.md  (skip if missing)
  'lab1.md',                   // → <board>/lab1.md
  'lab2.md',                   // → <board>/lab2.md
  'appendices/appendix-a.md',  // → appendices/appendix-a.md (always included)
],
```

### Step 2 — Add your content

Put your Markdown files in `lab_manuals/markdown/` matching the paths you listed in `pages`.

Put images in `lab_manuals/static/images/`. Reference them in Markdown as `/images/filename.png`.

### Step 3 — Run

```bash
cd pdf-gen
PUPPETEER_SKIP_DOWNLOAD=true npm install   # first time only
node generate.mjs
```

### Minimal single-PDF example

The smallest possible `config.mjs` with no multiplexing:

```js
export default {
  cover: {
    title:        'Getting Started with Zephyr',
    organization: 'Microhip Technology',
  },
  outputName: 'Zephyr-Getting-Started',
  versions: {
    boards: {
      same54: { displayName: 'SAME54 Xplained Pro' },
    },
    os: ['windows'],
  },
  introFile: 'home.md',
  pages: [
    'index.md',
    'lab1.md',
    'lab2.md',
    'lab3.md',
    'appendices/appendix-a.md',
  ],
};
```

This produces one PDF: `lab_manuals/pdf/same54/Zephyr-Getting-Started_same54_windows.pdf`.

## Part 2: Website (Docusaurus)

### Step 1 — Edit `gh_pages/docusaurus.config.js`

Update the site identity fields near the top:

```js
title: 'Getting Started with Zephyr',
url: 'https://your-github-username.github.io',
baseUrl: '/your-repo-name/',
organizationName: 'your-github-username',
projectName: 'your-repo-name',
```

Update the navbar items to match your sidebar sections:

```js
items: [
  { to: '/', label: 'Home', position: 'left' },
  { type: 'docSidebar', sidebarId: 'lab1Sidebar', label: 'Lab 1', position: 'left' },
  // add one entry per sidebar section
],
```

### Step 2 — Edit `gh_pages/sidebars.js`

Each sidebar section maps to a subdirectory under `lab_manuals/markdown/`. The sidebar is auto-generated from the files in that directory.

```js
const sidebars = {
  lab1Sidebar: [{ type: 'autogenerated', dirName: 'lab1' }],
  // add one entry per section
};
```

### Step 3 — Serve locally

```bash
cd gh_pages
npm install    # first time only
npm start      # opens localhost:3000
```

## Content Conventions

### Pure Markdown works fine

You do not need to use MDX, JSX components, or any of the advanced features. Plain `.md` files with standard Markdown work through the pipeline without any changes.

Advanced features that are available but entirely optional:

| Feature | Syntax | Purpose |
|---------|--------|---------|
| Admonitions | `:::note`, `:::warning`, etc. | Styled callout boxes |
| Variable substitution | `%BOARD%`, `%MY_VAR%` | Board/variant-specific text |
| OS tabs | `<Tabs><TabItem value="linux">` | Per-OS content variants |
| MDX component imports | `import X from './_part.md'` | Reusable content sections |
| Code annotations | `// highlight-next-line` | Line-level code highlighting in PDFs |

### Folder structure

The simplest structure is flat:

```
lab_manuals/
  markdown/
    intro.md
    lab1.md
    lab2.md
  static/
    images/
      diagram.png
```

For multiple boards, put board-specific files in named subdirectories:

```
lab_manuals/
  markdown/
    home.md            # appears in all PDFs
    shared/
      lab1.md          # appears in all PDFs (excluded from website sidebar)
    boardA/
      index.md         # appears only in Board A PDF
      lab2.md
    boardB/
      index.md         # appears only in Board B PDF
      lab2.md
  static/
    images/
```

> The `shared/` folder is excluded from the Docusaurus sidebar by default
> (configured in `docusaurus.config.js`). Files there are for PDF use only,
> or imported as components by other files.

### Variable substitution

If you define `vars` in `config.mjs`, any `%TOKEN%` in your Markdown is replaced at build time:

```js
// config.mjs
boards: {
  myboard: {
    displayName: 'My Board',
    vars: {
      BOARD_ID: 'myboard_v2',
      FLASH_CMD: 'west flash --runner openocd',
    },
  },
},
```

```markdown
<!-- lab1.md -->
Run: `west flash --board %BOARD_ID% %FLASH_CMD%`
```

Tokens that don't match any key are left unchanged.
